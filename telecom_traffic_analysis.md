# Анализ телекоммуникационного трафика

## 1. Цель анализа
Изучение структуры и уязвимостей сигнальных протоколов мобильных сетей (GTP, Diameter) на основе открытых pcap-файлов и документации.

## 2. GTP (GPRS Tunneling Protocol)
- **Назначение**: Туннелирование пользовательских данных (GTP-U) и управление сессиями (GTP-C) в сетях 3G/4G.
- **Пример трафика**: В открытых pcap-файлах (например, gtpv2.pcap с Wireshark Wiki) наблюдаются сообщения:
  - `Create Session Request`
  - `Modify Bearer Request`
- **Чувствительные данные**:
  - IMSI (International Mobile Subscriber Identity)
  - MSISDN (номер телефона)
  - APN (Access Point Name)
- **Уязвимости**:
  - Передача IMSI в открытом виде → отслеживание абонента.
  - Возможность подделки GTP-пакетов → DoS, перехват трафика.
  - Отсутствие аутентификации между узлами (в ранних версиях).

## 3. Diameter
- **Назначение**: Протокол AAA (Authentication, Authorization, Accounting) в сетях 4G (LTE).
- **Пример трафика**: В учебных pcap-файлах (например, с CloudShark) наблюдаются:
  - `Credit-Control-Request (CCR)`
  - `Authentication-Information-Request (AIR)`
- **Чувствительные данные**:
  - IMSI
  - Ключи аутентификации (в зашифрованном виде, но не всегда)
- **Уязвимости**:
  - Diameter **не шифруется по умолчанию** → MITM-атаки возможны.
  - Отсутствие строгой привязки к IP → подделка запросов.
  - Уязвимости в реализации на стороне оператора.

## 4. Сравнение с SS7
- **SS7** (2G/3G): ещё более уязвим — позволяет:
  - Отслеживать местоположение абонента.
  - Перехватывать SMS (включая 2FA-коды).
  - Прослушивать звонки.
- **Diameter/GTP** (4G/5G): более защищены, но **не идеальны**.

## 5. Рекомендации по защите
1. **Для операторов**:
   - Использовать **Diameter over TLS (DTLS)**.
   - Фильтровать GTP-трафик на границе сети (GTP firewall).
   - Мониторить аномалии (например, запросы из неожиданных стран).
2. **Для пользователей**:
   - Использовать приложения с end-to-end шифрованием (Signal, WhatsApp).
   - Избегать SMS-2FA — использовать authenticator-приложения или аппаратные ключи.

## 6. Вывод
Сигнальные протоколы мобильных сетей (SS7, Diameter, GTP) содержат критические уязвимости, позволяющие проводить слежку и перехват данных. Несмотря на улучшения в 4G/5G, полная безопасность не достигнута. Анализ трафика с помощью Wireshark остаётся ключевым инструментом для выявления рисков.
